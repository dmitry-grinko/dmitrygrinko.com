# 18. Зачем dataset делят на train / validation / test?

Dataset делят на несколько частей, чтобы получить **честную оценку качества модели** и избежать переобучения.

Если обучать модель и проверять её на одних и тех же данных, модель может просто запомнить dataset. В таком случае она будет показывать хорошие результаты на этих данных, но плохо работать на новых.

Поэтому данные разделяют на три части.

## Training set

Используется для обучения модели.

На этих данных алгоритм настраивает параметры модели и ищет закономерности.

## Validation set

Используется во время разработки модели.

С его помощью:

- сравнивают разные модели
- подбирают гиперпараметры
- контролируют переобучение

Модель не обучается напрямую на этих данных.

## Test set

Используется для финальной оценки модели.

Эти данные не участвуют ни в обучении, ни в настройке модели. Они позволяют понять, как модель будет работать на реальных данных.

## Итог

Dataset делят на несколько частей, чтобы:

- обучить модель
- настроить её параметры
- честно проверить качество на новых данных
