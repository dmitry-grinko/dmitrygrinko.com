# 47. Почему важно разделять эксперимент и production модель?

В машинном обучении обычно различают две стадии работы с моделью:

- **эксперименты (experiment / research)**
- **production**

Экспериментальная модель используется для разработки и тестирования идей. Production модель — это модель, которая уже используется в реальной системе.

Разделение этих двух этапов важно для стабильности и надёжности системы.

## Экспериментальная модель

На этапе экспериментов исследователь или инженер:

- пробует разные алгоритмы
- изменяет признаки
- тестирует гиперпараметры
- проверяет разные версии dataset

Такие модели могут часто меняться и не всегда быть стабильными.

## Production модель

Production модель — это версия модели, которая уже прошла проверку и используется в реальном продукте.

Она должна быть:

- стабильной
- воспроизводимой
- протестированной
- контролируемой

Любые изменения в production модели обычно проходят дополнительную проверку.

## Почему это важно

Если экспериментальные модели напрямую попадут в production, это может привести к проблемам:

- нестабильные результаты
- неожиданные ошибки
- ухудшение качества системы

Поэтому новые модели сначала тестируют в экспериментальной среде, а затем только после проверки развёртывают в production.

## Итог

Разделение экспериментальной и production моделей позволяет безопасно тестировать новые идеи и поддерживать стабильную работу реальной системы.
