# 50. Какие основные ошибки чаще всего делают начинающие в machine learning?

Начинающие в машинном обучении часто сталкиваются с похожими проблемами. Эти ошибки обычно связаны не столько с алгоритмами, сколько с данными, экспериментами и организацией процесса разработки.

## Игнорирование качества данных

Многие новички сразу начинают экспериментировать с моделями, не анализируя dataset.

Но если данные содержат:

- ошибки
- пропущенные значения
- шум
- сильный bias

то даже хорошая модель будет работать плохо.

## Отсутствие baseline

Иногда сразу начинают строить сложные модели, не имея простого baseline.

В результате трудно понять:

- действительно ли новая модель лучше
- есть ли реальный прогресс

Baseline помогает задать минимальный уровень качества.

## Неправильное разделение dataset

Распространённая ошибка — неправильное использование train, validation и test данных.

Например:

- обучение на test data
- использование test set для настройки модели

Это приводит к завышенным метрикам и недостоверной оценке модели.

## Data leakage

Иногда в training data случайно попадает информация, которая в реальной задаче недоступна.

Это делает результаты обучения нереалистично хорошими, но модель плохо работает в production.

## Переобучение

Начинающие часто используют слишком сложные модели для небольших datasets.

В результате модель:

- отлично работает на training data
- плохо работает на новых данных

## Игнорирование reproducibility

Если эксперименты нельзя воспроизвести, становится трудно:

- сравнивать модели
- повторять результаты
- находить ошибки

Поэтому важно фиксировать random seed, версии библиотек и конфигурацию экспериментов.

## Другие распространённые ошибки

Есть и другие проблемы, с которыми часто сталкиваются начинающие.

**Недостаточный анализ данных**

Иногда сразу начинают строить модели, не изучив dataset.
Но важно сначала понять данные:

- распределение признаков
- выбросы
- пропущенные значения
- корреляции

**Неправильный выбор метрик**

В некоторых задачах accuracy может вводить в заблуждение.
Например, в задачах обнаружения мошенничества или медицинской диагностики более полезны метрики вроде precision, recall или F1-score.

**Игнорирование роли признаков**

Новички часто концентрируются на выборе алгоритма, но качество модели во многих задачах сильнее зависит от правильно подготовленных признаков.

## Итог

Большинство ошибок начинающих в ML связано не с самими алгоритмами, а с:

- качеством данных
- неправильной оценкой моделей
- утечкой данных
- переобучением
- недостаточным контролем экспериментов
