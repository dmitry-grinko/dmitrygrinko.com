# 7. Что такое training data?

Training data (обучающие данные) — это набор примеров, который используется для обучения модели.

На этих данных модель ищет закономерности и настраивает свои параметры.

Каждый пример обычно состоит из двух частей:

- features — входные признаки
- label — правильный ответ (если задача обучается с учителем)

Общая структура выглядит так:

features → label

Например:

| площадь | комнаты | цена |
|-------|-------|------|
| 45    | 1     | 120000 |
| 60    | 2     | 170000 |
| 80    | 3     | 230000 |

Здесь:

- площадь и количество комнат — признаки
- цена — правильный ответ

Модель анализирует такие примеры и учится предсказывать цену для новых квартир.

## Минимальный пример обучения

Рассмотрим очень простой пример на Python с библиотекой scikit-learn (это популярная Python-библиотека с готовыми алгоритмами машинного обучения).

Также используется **NumPy** — библиотека для работы с числовыми массивами и таблицами данных.

В этом примере модель обучается предсказывать цену квартиры по площади.

```python
import numpy as np
from sklearn.linear_model import LinearRegression

# numpy используется для хранения данных в виде числовых массивов
X = np.array([
    [40],
    [60],
    [80],
    [100]
])

# y — правильные ответы (цены квартир)
y = np.array([
    110000,
    150000,
    210000,
    260000
])

# LinearRegression — модель линейной регрессии
model = LinearRegression()

# метод fit() обучает модель на training data
model.fit(X, y)

# predict() делает предсказание для новых данных
prediction = model.predict([[70]])

print(prediction)
```

Здесь:

- `X` — training data (признаки)
- `y` — правильные ответы
- `fit()` — запускает обучение модели

После обучения модель может делать предсказания для новых данных.

## Где берутся training data

В реальных системах обучающие данные обычно получают из:

- логов пользовательского поведения
- исторических данных компании
- размеченных датасетов
- ручной разметки данных

Например:

- изображения размечают люди (что на фото)
- письма отмечают как spam / not spam
- клики пользователей фиксируются автоматически

## Важный момент

Качество training data напрямую влияет на качество модели.

Если данные:

- маленькие
- шумные
- плохо размечены

то модель будет работать плохо независимо от алгоритма.

Поэтому в реальных ML-проектах большая часть работы связана именно с подготовкой данных.

## Итог

Training data — это данные, на которых модель учится.

Они содержат примеры входных данных и правильные ответы, которые позволяют алгоритму настроить параметры модели и научиться делать предсказания.
