# 4. Какие основные компоненты есть в любой ML-системе?

Несмотря на различия в алгоритмах и задачах, большинство ML-систем состоит из нескольких базовых компонентов.

## Данные

Это основа всей системы.

Данные используются для обучения модели и последующих предсказаний.

Обычно они включают:

- исходные признаки (features)
- иногда правильные ответы (labels)

Примеры данных:

- изображения
- текст
- события пользователей
- числовые таблицы

Качество данных напрямую влияет на качество модели.

## Feature engineering

Сырые данные редко подходят для обучения напрямую.

Поэтому их преобразуют:

- очищают
- нормализуют
- кодируют категории
- извлекают признаки

На этом этапе формируется набор **features**, который будет подаваться в модель.

## Модель

Модель — это математическая функция, которая принимает признаки и выдаёт предсказание.

Например:

- вероятность события
- класс объекта
- числовое значение

Тип модели зависит от задачи:

- линейная регрессия
- деревья решений
- градиентный бустинг
- нейронные сети

## Обучение (training)

Во время обучения алгоритм находит параметры модели, которые лучше всего объясняют данные.

Процесс обычно выглядит так:

1. модель делает предсказание
2. считается ошибка
3. параметры модели корректируются

Этот цикл повторяется много раз.

## Предсказания (inference)

После обучения модель используется для обработки новых данных.

Процесс простой:

```
features → модель → предсказание
```


Модель может работать:

- в API
- внутри приложения
- в batch-процессах
- в потоковых системах

## Мониторинг

После развёртывания модель нужно контролировать.

Проверяют:

- качество предсказаний
- изменения данных (data drift)
- скорость работы

Без мониторинга модель может постепенно деградировать.

## Итог

Любая ML-система обычно состоит из следующих компонентов:

- данные
- подготовка признаков
- модель
- обучение
- инференс
- мониторинг

Алгоритмы могут отличаться, но эта архитектура встречается почти во всех ML-системах.
