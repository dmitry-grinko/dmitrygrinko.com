# 50. Почему в machine learning часто говорят «garbage in, garbage out»?

Фраза "garbage in, garbage out" означает, что плохие данные приводят к плохим результатам.

Эта идея особенно важна в machine learning.

## Почему это происходит

Модель не понимает смысл данных.

Она просто анализирует закономерности.

Если dataset содержит:

- ошибки
- шум
- неправильные значения

модель будет строить закономерности на основе этой информации.

## Пример

Если dataset содержит неправильные labels, модель может научиться делать неправильные предсказания.

Даже если используется сложный алгоритм.

## Практический вывод

Качество модели редко определяется только алгоритмом.

Чаще всего результат зависит от:

- качества данных
- корректности preprocessing
- правильной подготовки dataset

## Итог

"Garbage in, garbage out" означает, что качество модели ограничено качеством данных, на которых она обучается.
