# 30. Как архитектура сети влияет на её возможности?

Архитектура сети определяет, **какие функции сеть вообще способна моделировать** и насколько сложные зависимости она может выучить.

Под архитектурой обычно понимают:

- количество слоёв  
- число нейронов в слоях  
- типы слоёв  
- типы активаций  
- способ соединения между слоями

## Если сеть слишком простая

Сеть может оказаться слишком слабой для задачи.

Например:

- мало слоёв  
- мало нейронов  
- ограниченная выразительность

В таком случае она может недообучиться.

## Если сеть сложнее

Больше слоёв и нейронов позволяют сети моделировать более сложные зависимости.

Это особенно важно в задачах:

- computer vision  
- NLP  
- speech recognition

## Но есть цена

Слишком сложная архитектура может:

- требовать больше данных  
- обучаться дольше  
- переобучаться  
- быть сложнее в настройке

## Пример

Небольшая сеть может быть достаточной для табличных данных.

Но для распознавания изображений часто нужны глубокие архитектуры, потому что задача сложнее.

## Итог

Архитектура сети определяет её выразительную силу, сложность обучения и риск переобучения.
