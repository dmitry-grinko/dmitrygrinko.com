# 49. Почему результаты обучения могут отличаться между запусками?

В машинном обучении один и тот же код может давать немного разные результаты при каждом запуске. Это нормально и связано с тем, что многие этапы обучения используют случайность.

## Основные причины

**Случайная инициализация параметров**

Многие модели начинают обучение со случайных начальных значений параметров.
Разные начальные точки могут приводить к немного разным итоговым решениям.

**Случайное разделение данных**

Dataset часто случайно делят на:

- training set
- validation set
- test set

Если разбиение меняется, модель обучается на немного разных данных.

**Случайный порядок данных**

Во время обучения данные часто перемешиваются (shuffle).
Это может немного менять траекторию обучения.

**Стохастические алгоритмы**

Некоторые алгоритмы используют случайность напрямую. Например:

- stochastic gradient descent
- dropout в нейронных сетях
- random forests

**Различия в вычислениях**

При работе на GPU или в распределённых системах порядок вычислений может отличаться, что иногда приводит к небольшим различиям в результатах.

## Как уменьшить различия

Чтобы сделать эксперименты более воспроизводимыми, обычно:

- фиксируют random seed
- используют одинаковое разделение dataset
- фиксируют версии библиотек
- сохраняют конфигурацию эксперимента

## Итог

Результаты обучения могут отличаться между запусками из-за случайности в алгоритмах, данных и вычислениях. Поэтому в ML важно контролировать условия эксперимента и обеспечивать reproducibility.
