# 38. Когда лучше удалить данные с пропущенными значениями?

Иногда удаление данных с пропущенными значениями может быть более разумным решением, чем их заполнение.

Это зависит от структуры dataset и количества пропусков.

## Когда пропусков мало

Если пропущенные значения встречаются редко, удаление строк обычно не влияет на dataset.

Например:

- dataset содержит 1 000 000 строк  
- пропуски есть только в 100 строках

Удаление таких строк практически не влияет на обучение.

## Когда признаки нерелевантны

Если признак содержит большое количество пропущенных значений и при этом мало влияет на модель, его можно удалить.

## Когда imputation может исказить данные

Иногда заполнение пропущенных значений может искажать распределение данных.

В таких случаях удаление может быть безопаснее.

## Итог

Удаление пропущенных данных оправдано, если:

- пропусков мало  
- признак не критичен  
- заполнение может искажать dataset
