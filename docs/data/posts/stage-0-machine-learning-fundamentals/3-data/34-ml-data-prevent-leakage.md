# 34. Как предотвратить data leakage?

Предотвращение data leakage — важная часть построения ML-систем.

Основная идея — убедиться, что модель **получает только ту информацию, которая будет доступна во время реального предсказания**.

## Корректное разделение dataset

Первое правило — разделить dataset на:

- train  
- validation  
- test

до выполнения любых операций обработки данных.

Все операции подготовки данных должны выполняться **только на training dataset**.

## Осторожность с feature engineering

Некоторые признаки могут случайно включать информацию о будущем.

Важно проверять:

- когда были собраны данные  
- какие признаки создаются  
- не содержит ли feature информацию о target

## Разделение по времени

В задачах с временными данными dataset часто делят по времени.

Например:

- старые данные — training  
- новые данные — test

Это помогает избежать использования информации из будущего.

## Итог

Чтобы избежать data leakage:

- корректно разделяйте dataset  
- внимательно проверяйте признаки  
- избегайте использования информации из будущего
