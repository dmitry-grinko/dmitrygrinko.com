# 2. Почему данные являются основой любой ML-модели?

В машинном обучении модель не содержит заранее прописанных правил.

В обычном программировании разработчик пишет логику:

if condition → действие

В machine learning правила появляются **из данных**.

Модель анализирует dataset и пытается найти закономерности между признаками и результатами.

Если данные плохие, модель не сможет построить правильные зависимости.

## Почему данные критичны

Качество модели напрямую зависит от данных:

- неправильные данные → неправильные выводы  
- шумные данные → нестабильная модель  
- маленький dataset → слабое обобщение

Даже очень сложная модель не сможет компенсировать плохие данные.

## Практическая реальность

В большинстве ML-проектов основная часть работы связана не с моделью, а с данными:

- сбор данных  
- очистка  
- подготовка  
- анализ распределений

Иногда на работу с данными уходит **80–90% времени проекта**.

## Итог

В machine learning модель учится на данных.  
Поэтому качество и структура данных напрямую определяют качество модели.
