# 33. Как обнаружить data leakage в dataset?

Data leakage не всегда очевиден, поэтому его обнаружение требует внимательного анализа данных и процесса подготовки dataset.

## Проверка признаков

Первый шаг — проверить, не содержат ли признаки информацию, напрямую связанную с target.

Например:

- признаки, созданные после события  
- признаки, содержащие результат

Если feature фактически содержит target, это явный признак leakage.

## Анализ слишком хороших метрик

Если модель показывает необычно высокие результаты:

- очень высокая accuracy  
- почти нулевая ошибка

это может быть сигналом leakage.

Особенно если задача объективно сложная.

## Проверка pipeline данных

Важно проверить весь pipeline подготовки данных:

- разделение dataset  
- нормализацию  
- генерацию признаков

Иногда leakage появляется именно на этапе preprocessing.

## Проверка временных данных

В задачах с временными рядами важно убедиться, что модель не использует информацию из будущего.

## Итог

Data leakage можно обнаружить через анализ признаков, проверку pipeline подготовки данных и подозрительно высокие метрики модели.
