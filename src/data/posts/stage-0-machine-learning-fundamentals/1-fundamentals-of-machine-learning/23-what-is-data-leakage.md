# 23. Что такое data leakage?

Data leakage — это ситуация, когда модель во время обучения получает информацию, которая в реальной задаче ей недоступна.

Из-за этого модель показывает очень хорошие результаты на тестовых данных, но плохо работает на новых данных в production.

Другими словами, модель **подсматривает правильный ответ** или использует данные, которые не должны участвовать в обучении.

## Пример

Представим модель, которая предсказывает, отменит ли клиент заказ.

В dataset есть признаки:

- время заказа
- тип товара
- способ оплаты
- статус возврата

Если в features случайно добавить **статус возврата**, модель почти идеально научится предсказывать отмену заказа, потому что этот признак напрямую связан с результатом.

Но в реальной системе такой информации заранее не существует.

Это и есть data leakage.

## Другой распространённый пример

Иногда утечка происходит из-за неправильного разделения данных.

Например:

- сначала данные нормализуют или агрегируют
- затем делят dataset на train и test

В этом случае информация из test data может попасть в training data.

## Почему это проблема

Data leakage делает оценку модели недостоверной:

- метрики выглядят очень хорошими
- кажется, что модель работает отлично
- на реальных данных качество резко падает

## Итог

Data leakage — это утечка информации из будущих или тестовых данных в процесс обучения модели, которая приводит к завышенной оценке качества модели.
