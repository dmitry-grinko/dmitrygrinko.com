# 40. Какие проблемы возникают при большом количестве категорий?

Когда категориальный признак содержит очень много уникальных значений, это может создавать несколько практических проблем при обучении моделей машинного обучения.

Такие признаки часто называют **high-cardinality features**.

Примеры таких признаков:

- city
- user_id
- product_id
- zip code

Количество уникальных категорий в них может достигать тысяч или даже миллионов.

## Резкий рост числа признаков

Если использовать one-hot encoding, каждая категория превращается в отдельную колонку.

Например:

- признак `city` содержит 1000 уникальных значений
- после one-hot encoding появится 1000 новых признаков

Это может сильно увеличить размер dataset и усложнить обучение модели.

## Разреженные данные

После one-hot encoding большинство значений в таблице становятся нулями.

Например:

| city_A | city_B | city_C |
|------|------|------|
| 1 | 0 | 0 |
| 0 | 1 | 0 |
| 0 | 0 | 1 |

Такие таблицы называют **sparse data**.

Разреженные данные могут:

- увеличивать потребление памяти
- замедлять обучение моделей

## Плохая обобщающая способность

Если некоторые категории встречаются очень редко, модель может не успеть выучить для них полезные закономерности.

В результате предсказания для редких категорий могут быть менее точными.

## Увеличение сложности модели

Большое количество признаков может привести к:

- увеличению времени обучения
- росту вычислительных затрат
- повышенному риску переобучения

## Итог

Большое количество категорий может приводить к:

- росту числа признаков
- появлению разреженных данных
- ухудшению обобщающей способности модели
- увеличению вычислительных затрат

Поэтому для таких признаков часто используют специальные методы кодирования или преобразования данных.
