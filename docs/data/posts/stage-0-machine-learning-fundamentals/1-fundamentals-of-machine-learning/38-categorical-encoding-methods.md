# 38. Какие способы кодирования категориальных признаков существуют?

Большинство моделей машинного обучения работают только с числовыми данными. Поэтому категориальные признаки (categorical features) обычно нужно преобразовать в числовой формат перед обучением модели.

Этот процесс называется **кодированием категориальных признаков (categorical encoding)**.

Существует несколько распространённых способов такого преобразования. Каждый из них подходит для разных типов данных и моделей.

## Label encoding

Label encoding — это самый простой способ кодирования категорий. Каждой категории присваивается уникальный числовой индекс.

Пример:

| device |
|------|
| mobile |
| desktop |
| tablet |

После кодирования:

| device |
|------|
| 0 |
| 1 |
| 2 |

Этот метод прост в реализации и почти не увеличивает размер dataset. Однако у него есть важное ограничение.

Числа могут создать **ложное ощущение порядка** между категориями. Например, модель может интерпретировать значение `2` как «больше» или «важнее» чем `1`.

Поэтому label encoding обычно используют только:

- для моделей на основе деревьев
- когда порядок категорий действительно имеет смысл

## One-hot encoding

One-hot encoding — один из самых распространённых методов кодирования категориальных признаков.

Каждая категория превращается в отдельный бинарный признак.

Пример исходных данных:

| device |
|------|
| mobile |
| desktop |

После кодирования:

| device_mobile | device_desktop |
|--------------|---------------|
| 1 | 0 |
| 0 | 1 |

Таким образом модель не воспринимает категории как упорядоченные значения.

Однако у этого метода есть недостаток: если категорий много, количество признаков может резко увеличиться. Это называется **curse of dimensionality**.

Например, если признак `city` содержит 1000 уникальных значений, после one-hot encoding появится 1000 новых колонок.

## Ordinal encoding

Ordinal encoding используется, когда категории имеют **естественный порядок**.

Пример:

| size |
|-----|
| small |
| medium |
| large |

После кодирования:

| size |
|-----|
| 1 |
| 2 |
| 3 |

Здесь порядок имеет смысл, поэтому модель может корректно использовать числовые значения.

Такие признаки часто встречаются в данных, где категории представляют уровни или ранги.

## Target encoding

Target encoding — более продвинутый способ кодирования.

Категория заменяется статистикой целевой переменной (target). Чаще всего используется **среднее значение label для данной категории**.

Пример:

| city | price |
|------|------|
| A | 200 |
| A | 220 |
| B | 150 |

После кодирования:

| city |
|------|
| 210 |
| 210 |
| 150 |

Этот метод особенно полезен, когда категорий очень много (например, тысячи уникальных значений).

Однако его нужно применять осторожно, потому что он может привести к **data leakage**, если кодирование выполняется неправильно.

## Как выбрать способ кодирования

Выбор метода зависит от нескольких факторов:

- типа модели
- количества категорий
- наличия естественного порядка между категориями
- размера dataset

Например:

- для линейных моделей часто используют **one-hot encoding**
- для деревьев решений иногда подходит **label encoding**
- для признаков с большим числом категорий может использоваться **target encoding**

## Итог

Категориальные признаки нельзя напрямую использовать в большинстве моделей машинного обучения. Поэтому их необходимо преобразовывать в числовой формат.

Наиболее распространённые методы кодирования:

- label encoding
- one-hot encoding
- ordinal encoding
- target encoding

Правильный выбор метода кодирования может существенно повлиять на качество модели.
