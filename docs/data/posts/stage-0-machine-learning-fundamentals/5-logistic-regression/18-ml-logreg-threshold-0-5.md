# 18. Почему часто используют threshold = 0.5?

Порог 0.5 используется по умолчанию, потому что он естественно разделяет вероятности на две группы.

## Интуитивная идея

Если вероятность больше 50%, модель считает, что объект скорее принадлежит положительному классу.

Если меньше 50%, вероятность отрицательного класса выше.

## Пример

P(class = 1) = 0.7

так как 0.7 > 0.5

модель выбирает класс 1.

## Почему это не всегда оптимально

В некоторых задачах ошибки разных типов имеют разную цену.

Например:

- пропустить мошенничество  
- ошибочно пометить нормальную транзакцию

В таких случаях threshold может изменяться.

## Итог

Threshold = 0.5 используется как простое и естественное правило для бинарной классификации.
