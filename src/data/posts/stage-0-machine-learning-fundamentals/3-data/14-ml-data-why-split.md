# 14. Почему dataset разделяют на train, validation и test?

Dataset разделяют на несколько частей, чтобы **корректно оценивать качество модели**.

Если обучать и проверять модель на одних и тех же данных, модель может просто запомнить dataset.

В этом случае метрики будут показывать хороший результат, но на новых данных модель будет работать хуже.

## Основная идея разделения

Разделение dataset позволяет решать разные задачи:

- train dataset — обучение модели  
- validation dataset — настройка модели  
- test dataset — финальная оценка

Каждая часть выполняет свою роль.

## Почему это важно

Такое разделение позволяет понять:

- насколько модель обобщает знания  
- не переобучилась ли модель  
- как она будет работать в реальных условиях

## Пример

Если dataset содержит 100 000 примеров, его можно разделить следующим образом:

- 70% — train  
- 15% — validation  
- 15% — test

## Итог

Dataset разделяют на части, чтобы обучать модель, настраивать её и объективно оценивать качество на новых данных.
