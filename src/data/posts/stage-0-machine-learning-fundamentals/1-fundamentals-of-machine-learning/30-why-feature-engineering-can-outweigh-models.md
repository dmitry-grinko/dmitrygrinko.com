# 30. Почему feature engineering может быть важнее модели?

В машинном обучении модель может находить закономерности только в тех признаках (features), которые ей передаются.

Если признаки плохо отражают структуру данных, даже сложная модель не сможет построить хорошее предсказание.

Поэтому во многих задачах качество признаков влияет на результат сильнее, чем выбор конкретного алгоритма.

## Пример

Представим задачу предсказания цены квартиры.

Dataset содержит только один признак:

| area | price |
|-----|------|
| 70  | 185000 |

Модель может использовать площадь, но ей трудно понять другие факторы, влияющие на цену.

Если добавить дополнительные признаки:

- количество комнат
- район
- возраст здания
- расстояние до центра

модель получает больше информации и может строить более точные предсказания.

## Почему это работает

Хорошие признаки помогают:

- лучше описать объект
- упростить задачу для модели
- выделить реальные закономерности в данных

Иногда простой алгоритм с хорошо подготовленными признаками работает лучше, чем сложная модель с плохими данными.

## Итог

Feature engineering может быть важнее модели, потому что именно признаки определяют, какую информацию модель получает из данных.
