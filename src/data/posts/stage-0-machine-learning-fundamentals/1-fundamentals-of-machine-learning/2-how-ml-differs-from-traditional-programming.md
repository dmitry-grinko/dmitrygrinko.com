# 2. Чем machine learning отличается от обычного программирования?

Главное отличие — где находится логика решения задачи.

В обычном программировании логика полностью задаётся разработчиком.
Программа работает строго по правилам, которые написаны в коде.

```
правила + данные → результат
```

Если поведение программы нужно изменить, меняется код.

В machine learning правила не пишутся вручную.
Алгоритм обучения извлекает их из данных и формирует модель.

```
данные + ответы → обучение → модель
модель + новые данные → результат
```

После обучения именно модель становится частью программы.

Это фундаментальное отличие архитектуры.

В обычной системе поведение определяется кодом.
В ML-системе поведение определяется параметрами модели, которые были получены из данных.

Из этого следуют важные инженерные последствия:

- качество системы зависит не только от алгоритма, но и от данных
- поведение модели может измениться после нового обучения
- систему приходится мониторить, потому что данные со временем меняются

Поэтому machine learning используют в задачах, где правила слишком сложны для ручного описания:

- распознавание изображений
- обработка текста
- рекомендации
- прогнозирование поведения пользователей

Если правила можно чётко сформулировать, обычное программирование обычно проще и надёжнее.

Machine learning имеет смысл там, где закономерности есть, но их трудно выразить в виде явного алгоритма.
