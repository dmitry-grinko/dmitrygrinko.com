# 31. Какие типы data leakage существуют?

Data leakage может возникать по-разному. Общая идея одна и та же: модель получает информацию, которая **не должна быть доступна во время обучения**.

На практике выделяют несколько распространённых типов leakage.

## Leakage из будущего (target leakage)

Это один из самых частых типов.

В dataset попадает информация, которая появляется **после события**, которое модель должна предсказать.

Пример:

Модель должна предсказывать, вернёт ли клиент кредит.

Если в dataset случайно попадает информация о том, что клиент уже допустил просрочку, модель будет использовать эту информацию.

Но в реальной системе она будет недоступна на момент предсказания.

## Leakage через preprocessing

Иногда leakage возникает из-за неправильной подготовки данных.

Например:

- нормализация выполнена на всём dataset  
- признаки рассчитаны с использованием test dataset

В этом случае модель получает информацию о тестовых данных.

## Leakage между train и test

Если training и test dataset не были корректно разделены, одинаковые или очень похожие объекты могут попасть в обе части.

Тогда модель фактически видит тестовые данные во время обучения.

## Итог

Основные типы data leakage:

- использование информации из будущего  
- ошибки в preprocessing  
- неправильное разделение dataset
