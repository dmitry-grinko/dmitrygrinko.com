# 25. Что такое bias в данных?

Bias в данных — это систематическое искажение dataset, из-за которого данные не отражают реальную ситуацию.

Это происходит, когда одни типы объектов представлены значительно чаще других или когда dataset собирался в ограниченных условиях. В результате модель обучается на неполной или перекошенной картине мира.

## Пример

Представим dataset для распознавания лиц.

Если большинство изображений в dataset — это люди одной возрастной группы или одной этнической группы, модель будет лучше распознавать именно эти лица и хуже работать для остальных.

Проблема возникает не из-за алгоритма, а из-за того, что training data не представляет всю реальную популяцию.

## Распространённые типы bias

В реальных datasets встречаются разные виды искажений.

**Sampling bias**

Возникает, когда данные собираются из ограниченной группы источников.

Например, если данные о пользователях собирались только из одной страны, модель может плохо работать для пользователей из других регионов.

**Measurement bias**

Появляется, когда данные измеряются или собираются разными способами.

Например, разные устройства могут по-разному измерять одно и то же значение.

**Historical bias**

Иногда dataset отражает прошлые решения или процессы. Если в данных уже присутствует перекос, модель может его воспроизводить.

## Почему это важно

Если dataset содержит bias, модель может:

- давать систематические ошибки
- хуже работать для определённых групп данных
- воспроизводить перекосы, которые есть в dataset

В машинном обучении часто действует простое правило:

```
garbage in → garbage out
```

Если данные искажены, модель будет повторять эти искажения.

## Итог

Bias в данных — это систематическое искажение dataset, при котором данные не отражают реальную ситуацию.

Из-за этого модель может хорошо работать только для части данных и хуже для остальных, даже если используется правильный алгоритм.
