# 22. Почему важно, чтобы training и test данные имели похожую distribution?

Модель машинного обучения обучается на **training data** и извлекает закономерности из этого набора данных. Эти закономерности отражают распределение (distribution) данных, на которых происходило обучение.

Если **test data имеет похожее распределение**, модель может применить те же закономерности и показать похожее качество.

Если же распределение сильно отличается, модель сталкивается с ситуацией, которой она не видела во время обучения.

```
training distribution ≠ test distribution → качество модели падает
```

## Пример

Представим модель, которая обучалась на данных о квартирах в небольшом городе.

Training data:

- площадь: 30–90 м²
- цена: 50k–150k

Если test data содержит квартиры:

- площадь: 150–300 м²
- цена: 400k–900k

то модель будет делать неточные предсказания, потому что такие данные она почти не видела во время обучения.

## Почему это проблема

Когда распределения отличаются, модель:

- не может корректно применять изученные закономерности
- начинает ошибаться чаще
- показывает более низкое качество

Эта ситуация называется **distribution shift**.

## Итог

Training и test данные должны иметь похожее распределение, чтобы:

- оценка модели была честной
- метрики отражали реальное качество
- модель могла применять изученные закономерности к новым данным
